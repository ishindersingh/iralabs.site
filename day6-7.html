Flashing the OS

Before your Raspberry Pi can think, it needs an operating system. We're installing Raspberry Pi OS (64-bit) onto the MicroSD card.

üì• What You Need

  • MicroSD Card (Class 10, minimum 16GB recommended)
  • SD Card Reader
  • PC/Laptop (Windows, Mac, or Linux)
  • Internet connection

üîß Step-by-Step Process

  1. Download Raspberry Pi Imager from raspberrypi.com/software
  2. Insert your MicroSD card into the reader
  3. Launch Raspberry Pi Imager
  4. Choose OS: Raspberry Pi OS (64-bit) - Recommended
  5. Choose Storage: Select your SD card
  6. Click the Gear Icon ‚öôÔ∏è for Advanced Settings
‚öôÔ∏è Critical Advanced Settings Configuration

This is where most students fail. Configure these settings before writing:

  • Hostname: iralab-robot.local
  • Enable SSH: ‚úÖ Use password authentication
  • Username: pi
  • Password: raspberry (or your choice)
  • Configure Wireless LAN: Enter WiFi SSID & Password
  • Wireless LAN Country: IN (India)
  • Locale Settings: Asia/Kolkata timezone

Click WRITE and wait for the process to complete (5-10 minutes). You'll see a verification step.

‚úÖ Success Indicator

When you see "Write Successful," safely eject the SD card.

2

Initial Boot & Network Discovery

üîå Power Up Sequence

  1. Insert the flashed MicroSD card into the Pi's slot
  2. Connect the USB Type-C power supply (5V, 3A minimum)
  3. The green LED will blink - this is normal
  4. Wait 2 minutes for first boot (filesystem expansion happens automatically)

üîç Finding Your Pi on the Network

You need to discover the Pi's IP address to connect via SSH. Use one of these tools:

üì± Mobile (Android/iOS)

Download Fing App from Play Store/App Store. Scan your network and look for "Raspberry Pi Foundation"

üíª Windows PC

Download Advanced IP Scanner. Run a scan and find "raspberry" in the manufacturer column

üçé Mac/Linux

Use Terminal command: arp -a or ping iralab-robot.local

üîê SSH Connection

Open your terminal (CMD on Windows, Terminal on Mac/Linux) and connect:

user@laptop:~$
ssh pi@192.168.x.x
# Replace 192.168.x.x with your Pi's actual IP
üîí First Connection Notice

You may see: "The authenticity of host can't be established"

Type yes and press Enter. Then enter your password: raspberry

‚úÖ You're In!

When you see pi@iralab-robot:~ $ you've successfully connected.

3

System Update & Permissions

A fresh OS is often outdated. We need to update system packages and configure proper hardware access permissions.

üì¶ Step A: Update System Packages

pi@iralab-robot:~ $
sudo apt update && sudo apt upgrade -y
# This may take 10-15 minutes. Be patient!

üë• Step B: Add User to Hardware Groups

Without proper group membership, you'll get "Permission Denied" errors when accessing GPIO pins.

pi@iralab-robot:~ $
sudo usermod -a -G gpio,i2c,spi,input pi
# Adds 'pi' user to all necessary hardware groups

‚öôÔ∏è Step C: Enable Hardware Interfaces

pi@iralab-robot:~ $
sudo raspi-config
üìã Inside raspi-config Menu
  1. Navigate to 3 Interface Options
  2. Enable the following:
    • I2C - For sensors (accelerometers, gyroscopes)
    • SPI - For displays and high-speed sensors
    • Serial Port - For GPS, Bluetooth modules
    • Camera - If using Pi Camera module
  3. Select Finish and choose Yes to reboot

After reboot, reconnect via SSH using the same command as before.

4

Understanding GPIO Layout

The Raspberry Pi has 40 pins. Not all are the same! Understanding this layout is critical for every circuit you build.

üìå 40-Pin GPIO Header Reference

üî¥ 5V Power (Pins 2, 4)
üü† 3.3V Power (Pins 1, 17)
‚ö´ Ground (Pins 6, 9, 14, 20, 25, 30, 34, 39)
üîµ GPIO Pins (0-27)

View full pinout at pinout.xyz or run pinout in terminal

‚ö†Ô∏è CRITICAL HARDWARE RULE

NEVER connect or disconnect wires while the Pi is powered ON!

A single loose 5V wire touching a GPIO pin will permanently destroy the CPU. Always:

  • Power OFF before wiring
  • Double-check connections
  • Use a multimeter to verify voltage levels
  • Keep wires organized and secured

üîå Common Pin Uses

GPIO 18 (Pin 12): PWM capable - perfect for LED dimming, servo control
GPIO 2 & 3 (Pins 3, 5): I2C interface (SDA & SCL) - for sensor communication
GPIO 14 & 15 (Pins 8, 10): UART TX/RX - for serial communication
5

Python Environment Setup

Modern Raspberry Pi OS (Bookworm) requires a virtual environment to install Python libraries. This prevents system conflicts.

üìÅ Create Project Workspace

pi@iralab-robot:~ $
mkdir ~/robot_ws && cd ~/robot_ws
# Creates a dedicated robotics workspace

üêç Setup Virtual Environment

pi@iralab-robot:~/robot_ws $
python -m venv env
# Creates isolated Python environment

source env/bin/activate
# Activates the environment (notice prompt changes to (env))

üìö Install Robotics Libraries

(env) pi@iralab-robot:~/robot_ws $
pip install RPi.GPIO spidev smbus2
# Core libraries for hardware control
üì¶ Library Breakdown
  • RPi.GPIO - Control GPIO pins, read sensors, drive motors
  • spidev - SPI communication for displays and high-speed devices
  • smbus2 - I2C communication for sensors (IMU, temperature, etc.)

Pro Tip: Always activate your environment before working: source ~/robot_ws/env/bin/activate

6

The "Alive" Test - LED Blink

Time to bring your Pi to life! This test verifies your permissions, wiring, and Python setup.

üîå Hardware Setup

Wiring Instructions:
  1. Power OFF the Pi
  2. Connect LED positive leg (longer) to GPIO 18 (Pin 12)
  3. Connect 220Œ© resistor to LED negative leg
  4. Connect resistor other end to Ground (Pin 6)
  5. Double-check all connections
  6. Power ON the Pi

üíª Create Your First Script

(env) pi@iralab-robot:~/robot_ws $
nano blink.py

Paste this code into the nano editor:

import RPi.GPIO as GPIO
import time

LED_PIN = 18
GPIO.setmode(GPIO.BCM) # Use BCM pin numbering
GPIO.setup(LED_PIN, GPIO.OUT)

print("ü§ñ System Online. Blinking LED on Pin 18...")
try:
    while True:
        GPIO.output(LED_PIN, True) # LED ON
        time.sleep(0.5)
        GPIO.output(LED_PIN, False) # LED OFF
        time.sleep(0.5)
except KeyboardInterrupt:
    GPIO.cleanup() # Clean exit - ALWAYS do this!
    print("\n‚úÖ Clean Exit Successful")

Save and exit: Press Ctrl + X, then Y, then Enter

‚ñ∂Ô∏è Run Your First Hardware Script

(env) pi@iralab-robot:~/robot_ws $
python blink.py
‚úÖ Success!

Your LED should be blinking every 0.5 seconds. Press Ctrl + C to stop the program gracefully.

üéì What You Just Learned

  • GPIO.setmode(GPIO.BCM) - Use Broadcom pin numbering (not physical pin numbers)
  • GPIO.setup() - Configure pin as input or output
  • GPIO.output() - Set pin HIGH (3.3V) or LOW (0V)
  • GPIO.cleanup() - Reset all pins to safe state (prevents hardware damage)
  • try/except KeyboardInterrupt - Handle Ctrl+C gracefully

Troubleshooting Common Issues

‚ùå Problem: "Permission Denied" Error

Solution: Your user isn't in the GPIO group yet. Run these commands:

sudo chown root.gpio /dev/gpiomem
sudo chmod g+rw /dev/gpiomem
# Then logout and login again, or reboot

‚ùå Problem: LED Not Blinking

Check These:

  • Is the LED connected in the correct polarity? (Long leg to GPIO)
  • Is the resistor connected? (Without it, LED may burn out)
  • Did you use the correct pin? (GPIO 18 is Physical Pin 12)
  • Run pinout command to verify pin layout

‚ùå Problem: "No module named RPi.GPIO"

Solution: You forgot to activate your virtual environment!

source ~/robot_ws/env/bin/activate
# Your prompt should now show (env) prefix

‚ùå Problem: Can't SSH to Pi

Troubleshooting Steps:

  1. Verify Pi and laptop are on the same WiFi network
  2. Check if green LED is blinking (indicates boot activity)
  3. Try hostname: ssh pi@iralab-robot.local
  4. Re-flash SD card with SSH enabled in advanced settings
  5. Connect monitor + keyboard directly to Pi as last resort
